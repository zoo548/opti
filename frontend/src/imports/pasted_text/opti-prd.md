프로젝트명 Opti
플랫폼 반응형 웹 (Mobile-first → Desktop 대응, Vercel 배포)

1. 개요
1.1 제품 한 줄 설명
출발지·도착지를 입력하고 희망 도착시각 또는 가격 범위를 선택하면, 파레토 프론티어 기반으로 최적 택시 전환 지점을 순위별로 추천하는 반응형 웹 서비스.
1.2 문제 정의
수도권 통근자는 대중교통(저비용·장시간)과 택시(고비용·단시간)라는 양극단의 선택지만 갖는다. 현재 네이버·카카오맵 등 어떤 경로 탐색 서비스도 두 수단을 조합한 하이브리드 경로, 즉 "어디서 택시로 갈아타면 최적인가"를 제안하지 않는다.
1.3 목표

주어진 제약(시간 또는 비용) 안에서 가중치 누적시간·비용을 동시에 최소화하는 파레토 최적 환승 지점을 도출한다.
결과를 사용자가 직관적으로 비교·선택할 수 있도록 반응형 UI로 제공한다.


2. 사용자
2.1 타깃

수도권 통근·통학자 (특히 대중교통이 우회 구조인 지역 거주자)
시간과 비용을 동시에 고려해 이동 수단을 선택하고 싶은 사용자

2.2 핵심 사용 시나리오

혜성은 경기도 집에서 서울대학교까지 출근한다.
지금 시각은 8:10이고 9시까지 도착하고 싶다.
Opti에 출발지·도착지를 입력하고 도착 희망 시각 "09:00"을 입력하면
시스템이 허용 소요시간 50분을 계산하고,
"양재역 환승 — 총 48분 / 12,400원"이 1순위로 추천된다.
카드를 보면 "지하철 2호선 25분 → 양재역 환승 → 택시 23분" 구간이 바로 보인다.

4.3 P3 — 결과 목록
